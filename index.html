Chapter 1: Register Transfer & Micro-operations
// BASICS TO ADVANCED — EVERYTHING YOU NEED
1.1 What is a Register?

A register is a small, fast storage unit inside the CPU made of flip-flops. Each flip-flop stores 1 bit. An n-bit register stores n bits of data.

Think of it like a tiny whiteboard inside the processor where temporary calculations are written. Unlike RAM (which is far away and slow), registers are built right into the chip and work at CPU speed.

💡 Key Registers in Basic Computer DR (Data Register, 16-bit), AR (Address Register, 12-bit), AC (Accumulator, 16-bit), IR (Instruction Register, 16-bit), PC (Program Counter, 12-bit), TR (Temporary Register, 16-bit), INPR (Input Register, 8-bit), OUTR (Output Register, 8-bit)
1.2 Register Transfer Language (RTL)

RTL is a symbolic notation used to describe how data moves between registers. It's like a math formula for hardware operations.

📝 RTL Notation Rules R1 ← R2    (Copy contents of R2 into R1)
R1 ← R1 + R2    (Add R1 and R2, store in R1)
If (T=1): R1 ← R2    (Conditional transfer)
M[AR] ← R1    (Store R1 into memory at address in AR)
R1 ← M[AR]    (Load from memory into R1)

The arrow ← means "gets" or "is loaded with". Operations inside register-shaped brackets happen simultaneously in hardware.

1.3 Bus and Memory Transfer

A bus is a group of wires shared by many registers. Instead of wiring every register directly to every other register (which would be too many wires!), we connect all registers to one common bus.

Only ONE register can place data on the bus at a time (controlled by select signals), but any register can receive from the bus simultaneously.

Fig 1.1 — Common Bus connecting all registers (data flows through the shared bus)
🔧 How a MUX-based Bus Works (4 registers) A 4×1 MUX selects which register drives the bus. Selection is via S1, S0 bits.
S1 S0 = 00 → Register A on bus
S1 S0 = 01 → Register B on bus
S1 S0 = 10 → Register C on bus
S1 S0 = 11 → Register D on bus
1.4 Arithmetic Micro-operations

These are basic math operations performed on binary numbers stored in registers.

SubtractR0 ← R1 + R2' + 1Using 2's complement subtraction
IncrementR1 ← R1 + 1Add 1 to R1

NegateR1 ← R1' + 12's complement of R1

Binary Adder-Subtractor Circuit

A 4-bit adder-subtractor uses one circuit to do both addition and subtraction. The trick: to subtract B from A, we add A + (2's complement of B) = A + B̄ + 1.

A single XOR gate and a mode bit M controls this: when M=0, the circuit adds (XOR outputs B unchanged). When M=1, the circuit subtracts (XOR inverts B, and M=1 feeds into carry-in, adding 1 automatically).

1.6 Arithmetic Logic Shift Unit (ALU)

The ALU is the heart of the CPU. It combines an arithmetic unit, a logic unit, and a shift unit into one circuit controlled by selection inputs (S3, S2, S1, S0) and a mode bit.

The control word determines which operation is performed. In general register organization, a 4-bit control input selects from 16 arithmetic operations and 16 logic operations.

Chapter 2: Basic Computer Organization & Design
// INSTRUCTION CYCLE · I/O · ACCUMULATOR LOGIC
2.1 Instruction Codes

An instruction tells the computer what to do. In the basic computer, each instruction is 16 bits long:

Instruction Format (16-bit) Bits 15–12: Opcode (4 bits → 16 possible instructions)
Bit 15: If 1 and opcode = 111, it's a Register Reference or I/O instruction
Bit 12: Addressing mode (0 = Direct, 1 = Indirect)
Bits 11–0: Address (12 bits → can address 4096 memory locations)

There are 3 types of instructions: Memory Reference (MRI), Register Reference (RRI), and Input-Output (IOI).

2.2 Memory Reference Instructions (MRI)

These instructions access memory. The address field points to a memory location.

MnemonicOpcodeOperationRTL
AND000AND memory to ACAC ← AC ∧ M[EA]
ADD001Add memory to ACAC ← AC + M[EA], E←carry
LDA010Load AC from memoryAC ← M[EA]
STA011Store AC to memoryM[EA] ← AC
BUN100Branch unconditionallyPC ← EA
BSA101Branch and save return addrM[EA]←PC, PC←EA+1
ISZ110Increment and skip if zeroM[EA]←M[EA]+1; if M[EA]=0: PC←PC+1
⚡ Memory: Direct vs Indirect Addressing Direct: The address in the instruction IS the memory address. EA = address field.
Indirect: The address in the instruction points to a memory location that CONTAINS the actual address. EA = M[address field].
2.3 Instruction Cycle

The instruction cycle is the step-by-step process the CPU repeats for every instruction. It has 4 phases:

1. Fetch — Get the instruction from memory into IR.
2. Decode — Decode what instruction it is (opcode).
3. Execute — Perform the operation.
4. Check for Interrupt — Check if an interrupt occurred.

Fig 2.1 — Complete Instruction Cycle Flowchart
2.4 Interrupt Cycle

An interrupt is a signal that temporarily stops the current program to handle a high-priority event (like input from keyboard or output to printer becoming ready).

When an interrupt occurs (IEN=1 and FGI or FGO=1), the CPU:
1. Saves the return address (PC) at address 0 of memory: M[0] ← PC
2. Loads address 1 into PC to jump to interrupt service routine: PC ← 1
3. Disables further interrupts: IEN ← 0, R ← 0

Fig 2.2 — Interrupt Cycle Flowchart
2.5 Input-Output Configuration

The basic computer communicates with I/O devices through flags and registers:

I/O Flags and Registers FGI (Input Flag): Set to 1 when input device has data ready. CPU reads INPR.
FGO (Output Flag): Set to 1 when output device is ready to accept data. CPU writes OUTR.
INPR: 8-bit register holding incoming data from keyboard.
OUTR: 8-bit register holding data to send to display.
Fig 2.3 — Input-Output Configuration of Basic Computer
2.6 Accumulator Logic Design

The accumulator (AC) is the main working register of the CPU. It holds intermediate results. The accumulator logic receives inputs from the ALU, INPR, and can be loaded, cleared, complemented, or shifted.

Operations on AC AC ← 0 (Clear), AC ← AC' (Complement), AC ← DR (Load), AC←AC+DR (Add), shl AC (Shift Left), shr AC (Shift Right), AC(0-7) ← INPR (Input)

3.1 Machine Language vs Assembly Language

Machine Language: Instructions in pure binary (0s and 1s). Humans can't read it easily. Every instruction is a binary number.

Assembly Language: Uses mnemonics (short text codes like LDA, ADD, STA) to represent instructions. Much easier for humans to write. An Assembler converts assembly to machine code.

3.2 Assembler — First Pass & Second Pass

An assembler is a program that converts assembly language code into machine code (binary). It makes two passes over the source code.

First Pass — Build the Symbol Table

The first pass scans through the entire program and builds a Symbol Table — a list of all labels and their memory addresses. It does NOT generate machine code yet.

Fig 3.1 — Assembler First Pass Flowchart

Second Pass — Generate Machine Code

The second pass uses the Symbol Table built in pass 1 to translate each instruction into its binary machine code equivalent.

Second Pass Steps 1. Read each line of source code again
2. Look up opcodes from an opcode table
3. Look up label addresses from symbol table
4. Generate binary machine code for each instruction
5. Write the machine code to output file
3.3 Subroutines

A subroutine is a separate section of code that performs a specific task, and can be called multiple times from different places in the program. This avoids repeating the same code.

How Subroutines Work 1. Main program executes BSA instruction (Branch and Save Address)
2. BSA saves the return address (next PC value) at the first address of the subroutine
3. Control jumps to subroutine body (subroutine address + 1)
4. Subroutine executes
5. At the end, BUN M[sub_address] restores the saved return address and jumps back
/ Main program calling a subroutine ORG 100 LDA X / Load X BSA SUB / Call subroutine (saves PC=104 at SUB) STA Y / Resume here after return HLT / Subroutine at address SUB SUB, HEX 0 / This location stores return address CMA / Complement AC (body of subroutine) INC / Increment BUN SUB / Indirect branch back to saved address
3.4 Pseudo Instructions

Pseudo instructions are assembler directives — they are not real machine instructions. They give directions TO the assembler, not to the CPU.

Pseudo InstructionMeaningExample

A control word (14 bits) specifies the operation:

4.2 Stack Organization

A stack is a Last-In-First-Out (LIFO) data structure. Think of it like a stack of plates — you add and remove from the top only.

The Stack Pointer (SP) always points to the top of the stack. Two main operations: PUSH (add item) and POP (remove item).

64-Word Register Stack SP (6-bit) points to top of stack (0 to 63).
FULL flag: Set to 1 when stack is full (SP=63 after push)
EMTY flag: Set to 1 when stack is empty (SP=0 after pop)

PUSH operation: SP ← SP+1; M[SP] ← DR; if SP=63: FULL←1; EMTY←0
POP operation: DR ← M[SP]; SP ← SP-1; if SP=0: EMTY←1; FULL←0

Q1A / Q2B Explain shift micro-operations and draw 4-bit combinational circuit shifter. [5 Marks]
🔥 6x Asked
▼
✅ Detailed Answer

Shift Micro-operations

Shift micro-operations shift the bits of a register to the left or right. They are of three types:

1. Logical Shift: In logical shift left (shl), bits move left; 0 is inserted at the LSB, and the MSB is lost. In logical shift right (shr), bits move right; 0 is inserted at the MSB, and the LSB is lost.

2. Circular Shift (Rotate): In circular shift left (cil), the MSB that is shifted out wraps around to the LSB position. In circular shift right (cir), the LSB wraps to the MSB. No bit is lost.

3. Arithmetic Shift: Arithmetic shift right (ashr) preserves the sign bit (MSB). The sign bit is copied into the new MSB instead of inserting 0. This is used for signed division by 2. Arithmetic shift left is same as logical shift left.

OperationSymbolRTLBit entering
Logical Shift Leftshl RR←R*2 (multiply by 2)0 at LSB
Logical Shift Rightshr RR←R/2 (divide by 2)0 at MSB
Circular Leftcil RMSB wraps to LSBMSB → LSB
Circular Rightcir RLSB wraps to MSBLSB → MSB
Arithmetic Rightashr RSign bit replicatedMSB copied

4-bit Combinational Circuit Shifter

This circuit performs shift-left or shift-right using four 2-to-1 multiplexers and one selection input S.

When S=0 (No Shift): Each MUX passes the input bit straight through (Ai → Yi).

When S=1 (Shift Left): MUX0 takes input from serial-in-left (SIL), MUX1 takes A0, MUX2 takes A1, MUX3 takes A2. Thus Y3=A2, Y2=A1, Y1=A0, Y0=SIL.

Q1B / Q2B Explain Binary Adder-Subtractor with its Circuit diagram. [5 Marks]
🔥 6x Asked
▼
✅ Detailed Answer

A Binary Adder-Subtractor is a combinational circuit that can perform both addition and subtraction of two 4-bit numbers using a single circuit.

Key Concept — 2's Complement Subtraction

A - B = A + (2's complement of B) = A + B̄ + 1

So to subtract, we just need to: (1) complement all bits of B (using XOR gates), and (2) add 1 (by setting the initial carry-in to 1).

Circuit Operation

A mode bit M controls the operation. Each bit of B passes through an XOR gate with M:

When M=0: XOR(Bi, 0) = Bi → B unchanged → performs ADDITION (A+B). Carry-in to FA0 = 0.

When M=1: XOR(Bi, 1) = Bi' → B complemented → combined with carry-in=1 (M itself) → performs SUBTRACTION (A-B = A + B̄ + 1).

MInput to FACin of FA0Result
0Bi (unchanged)0A + B (Addition)
1Bi' (complemented)1A + B̄ + 1 = A - B (Subtraction)

Overflow: If the result exceeds the range of 4 bits, an overflow (V) is detected: V = C4 XOR C3 (where C4 is final carry out and C3 is carry into last adder).

The circuit diagram was shown above in the notes section (Fig 1.2). The 4 XOR gates handle the mode control, and 4 full adders chain their carry outputs to produce the final sum.

Q1A Design and explain a common bus system for four registers using MUX. [5 Marks]
🔥 6x Asked
▼
✅ Detailed Answer

A common bus system allows multiple registers to share a single set of wires (the bus) for data transfer. Instead of connecting every register directly to every other register (which would require n×(n-1) connections), only one bus is needed.

Design Using Multiplexers (MUX)

For 4 registers (A, B, C, D), each of n-bits wide: We need n multiplexers, one for each bit position. Each MUX is a 4-to-1 MUX that selects which register drives that bit line of the bus. Two selection bits S1 and S0 choose the register.

S1S0Register on Bus
00Register A
01Register B
10Register C
11Register D

Loading: Any register can receive data from the bus by enabling its load input. This is typically controlled by separate enable signals (LD for each register).

Three-state bus buffers are an alternative to MUX: each register has a three-state buffer on its output. Only the selected register's buffer is enabled (output = data), others are in high-impedance state (effectively disconnected). This avoids MUX hardware but requires careful control to prevent bus contention.

Q2B Explain the design of one stage of Arithmetic Logic Shift Unit. [5 Marks]
4x Asked
▼
✅ Detailed Answer

The Arithmetic Logic Shift Unit (ALSU) combines all micro-operations into one circuit. One stage of the ALSU processes 1 bit of a multi-bit operation.

One Stage Design

Each stage has three sub-units feeding into a 4:1 MUX:

Arithmetic unit stage: A Full Adder (FA) takes: input A(i), B(i) or B'(i), and carry-in from previous stage. It produces sum S(i) and carry-out to next stage.

Logic unit stage: Takes A(i) and B(i) and performs one of 4 logic operations selected by S1, S0: AND (00), OR (01), XOR (10), NOT A (11).

Shift unit stage: Takes A(i-1) (shift left) or A(i+1) (shift right) depending on direction.

The final 4:1 MUX (selection inputs S3, S2) chooses which unit's output to pass to Yi: 00=Arithmetic, 01=Logic, 10=Shift Left, 11=Shift Right.

Register Transfer Language (RTL) is a symbolic notation used to describe the movement of data between registers and the operations performed on that data in digital systems. It provides a concise way to represent hardware behavior using mathematical notation.

Key RTL Symbols

SymbolMeaningExample
←Transfer (copy) dataR2 ← R1 (copy R1 to R2)

Register Transfer Process

A register transfer requires: (1) a source register whose contents are placed on a bus, (2) a bus to carry the data, and (3) a destination register that loads the data when a control signal (clock pulse) activates it.

The operation R1←R2 means: the output of R2 is connected to the input of R1. A timing control signal (usually a clock edge) triggers R1 to capture the value. R2 is not changed — its value is only READ.

Q1A Explain Binary Incrementer. [5 Marks]
2x Asked
▼
✅ Detailed Answer

A Binary Incrementer adds 1 to a binary number. It uses half adders to implement R ← R + 1.

For a 4-bit incrementer: the carry-in to the first half adder is 1 (hardwired). Each subsequent half adder takes a register bit and the carry from the previous stage.

A half adder has: Sum = A XOR B, Carry = A AND B.

HA0: X0=A0⊕1=A0', C0=A0 (only carry when A0=1)
HA1: X1=A1⊕C0, C1=A1·C0
HA2: X2=A2⊕C1, C2=A2·C1
HA3: X3=A3⊕C2, C3=A3·C2 (overflow bit)

Q1A / Q2B What is an Instruction Cycle? Draw and explain flowchart of it. [5 Marks]
🔥 7x Asked
▼
✅ Detailed Answer

An Instruction Cycle is the complete process that the CPU goes through repeatedly to execute each instruction. Every program instruction goes through this cycle.

Phases of Instruction Cycle

Phase 1 — FETCH: Get the instruction from memory into the processor.

T0: AR ← PC // Copy program counter to address register T1: IR ← M[AR], PC←PC+1 // Load instruction, increment PC

Phase 2 — DECODE: Figure out what instruction was fetched.

T2: AR ← IR(11-0), CAR ← D // Decode opcode in IR(15-12) // D0..D7 = decoder outputs, CAR = control address register

Phase 3 — DETERMINE EFFECTIVE ADDRESS: If instruction is indirect mode, one more memory access is needed.

T3: If (I=1 AND D7=0): AR ← M[AR] // Indirect → fetch actual address

Phase 4 — EXECUTE: Carry out the instruction's operation (depends on opcode).

Phase 5 — CHECK INTERRUPT: After execution, check if an interrupt needs to be handled before the next instruction.

If (IEN=1) AND (FGI=1 OR FGO=1): R←1 // Set interrupt request

See the complete flowchart diagram in Chapter 2 notes section above (Fig 2.1).

Key timing signals: The basic computer uses timing signals T0–T7 generated by a sequence counter. Each timing signal enables one micro-operation.
Q2B / Q3A Draw and explain the flowchart for Interrupt Cycle. [5 Marks]
🔥 4x Asked
▼
✅ Detailed Answer

The Interrupt Cycle is the CPU's response to an interrupt request (R=1). It saves the current execution state and branches to the interrupt service routine (ISR).

Micro-operations in Interrupt Cycle

T0: AR ← 0, TR ← PC // AR points to memory location 0; TR saves return address T1: M[AR] ← TR, PC ← 0 // Save return address at M[0]; PC jumps to location 0 T2: PC ← PC+1, IEN ← 0, R←0, SC←0 // PC becomes 1 (ISR starts at addr 1); disable further interrupts; reset R and SC

After this, the instruction cycle continues from the ISR. At the end of the ISR, an ION instruction re-enables interrupts and a BUN instruction (indirect) reads M[0] to get the return address and goes back to the original program.

The interrupt cycle flowchart was shown in Fig 2.2 in the notes. The diamond checks if R=1; if yes, executes the micro-operations above; if no, the fetch cycle begins immediately.

Why disable interrupts (IEN←0)? To prevent another interrupt from occurring while the first one is being serviced, which could corrupt the saved return address at M[0].

Q2A Explain Input and Output configuration with suitable block diagram. [5 Marks]
4x Asked
▼
✅ Detailed Answer

The basic computer uses a program-controlled I/O system where the CPU checks flags to coordinate data transfer with I/O devices.

Input Configuration

The keyboard is the input device. It connects to INPR (8-bit Input Register) and FGI (1-bit Input Flag). When a key is pressed: the 8-bit ASCII code goes to INPR and FGI is set to 1. The CPU periodically checks FGI; when FGI=1, it reads INPR into AC using SKI + INP instructions. After reading, FGI is cleared to 0.

Output Configuration

The display/printer is the output device. OUTR (8-bit Output Register) and FGO (1-bit Output Flag) manage it. Initially FGO=1 (device ready). CPU checks FGO; when FGO=1, it places data in OUTR using OUT instruction, then FGO is cleared to 0. When the device finishes printing, FGO is set back to 1.

AND DR with ACAC←AC∧DRAND instruction
Clear ACAC←0CLA instruction
Complement ACAC←AC'CMA instruction
Shift left ACAC←shl(AC)CIL instruction
Shift right ACAC←shr(AC)CIR instruction
Load from INPRAC(0-7)←INPRINP instruction

Hardware Design

The accumulator register has a parallel-load capability controlled by LD(AC) signal. It also has clear (CLR(AC)) and increment (INR(AC)) inputs. The input to AC comes from an OR gate that selects between ALU output, INPR, and other sources based on the current operation's control signals.

The adder-and-logic unit (ADDER AND LOGIC) receives AC and DR, performs the operation, and feeds result back to AC input. An 8:1 MUX or equivalent combinational logic determines which operation's result is loaded into AC.

Q2B Explain the operation of three-state bus buffers and their use in common bus design. [5 Marks]
2x Asked
▼
✅ Detailed Answer

A three-state buffer (tri-state buffer) has three output states: logic 0, logic 1, and high-impedance (Z) state. When enabled (E=1), it acts as a regular buffer (output = input). When disabled (E=0), the output is in high-impedance state — it appears disconnected from the bus, as if the wire doesn't exist.

This is perfect for bus design: multiple registers can connect to the same bus wires. Only ONE register enables its tri-state buffers at a time. All others remain in high-Z, so they don't interfere. This avoids the short-circuit that would occur if two registers tried to drive the bus simultaneously.

Comparison: MUX vs Three-state bus MUX: Routes signals through multiplexer hardware. Requires large MUX (4:1 for 4 registers × n bits).
Three-state: Each register connects directly to bus via buffers. Simpler wiring, but needs enable control. Slight propagation delay. Both achieve the same function of allowing one-at-a-time bus access.

Q3A Define Assembler and explain First Pass of an assembler with flow chart. [5 Marks]
🔥 7x Asked
▼
✅ Detailed Answer

An Assembler is a system program that translates assembly language programs into machine language (binary) programs. It acts as a translator between human-readable mnemonics and binary codes the hardware can execute.

Why Two Passes?

In a single pass, when the assembler encounters a forward reference (a label used before it is defined), it cannot know the address. The two-pass approach solves this: Pass 1 builds the complete symbol table first, then Pass 2 can resolve all references.

First Pass — Detailed Steps

Step 1: Initialize Location Counter (LC) to 0. LC keeps track of the memory address of each instruction.

Step 2: Read the first/next line of assembly code.

Step 3: Check if it's an ORG pseudo-instruction. If yes, set LC to the ORG value and go back to Step 2.

Step 4: Check if the line has a label (name before the comma). If yes, store the label and current LC value in the Symbol Table.

Step 5: Increment LC by 1 (each instruction occupies one memory word).

Step 6: Check if it's the END pseudo-instruction. If yes, stop Pass 1. If not, go to Step 2.

See Fig 3.1 in Chapter 3 notes for the complete flowchart.

Example Symbol Table after Pass 1:
Source:   ORG 100 / LDA A / ADD B / STA C / HLT / A, DEC 5 / B, DEC 3 / C, DEC 0 / END
Symbol Table: A→104, B→105, C→106
Q3A Define Assembler and explain Second Pass of an assembler with flow chart. [5 Marks]
🔥 4x Asked
▼
✅ Detailed Answer

The Second Pass of the assembler uses the Symbol Table (built in Pass 1) to translate each assembly instruction into its binary machine code equivalent.

Second Pass Steps

Step 1: Initialize LC=0 again. Open output file for machine code.

Step 2: Read next line of source code.

Step 3: Check for ORG — if found, set LC to specified value.

Step 4: If line has an operand symbol (label), look it up in the Symbol Table to get its address.

Step 5: Look up the mnemonic in the Machine Operation Table (MOT) to get the binary opcode.

Step 6: Combine the opcode and address (and addressing mode bit I) to form the 16-bit machine instruction. Write it to the output at location LC.

Step 7: Increment LC. If not END, go to Step 2.

Example (continuing from Pass 1):
LDA A → Look up: LDA=010, A=104(decimal)=001101000(binary) → 0 010 0 001101000100
ADD B → opcode 001, B=105 → binary instruction
STA C → opcode 011, C=106 → binary instruction
Q3B Explain Memory stack in detail / Stack organization. [5 Marks]
3x Asked
▼
✅ Detailed Answer

A Memory Stack is a portion of main RAM organized and accessed in LIFO (Last-In-First-Out) order. It is pointed to by a Stack Pointer (SP) register.

Memory Stack vs Register Stack

In a register stack (like 64-word stack), all storage elements are registers built inside the CPU — fast but expensive and limited in size.

In a memory stack, the stack occupies a region of main RAM. The SP register holds the address of the current top of the stack. This allows a much larger stack (limited only by RAM size) but is slower since RAM access is needed for every push/pop.

PUSH Operation (memory stack):

SP ← SP - 1 // Decrement SP (stack grows downward in memory) M[SP] ← DR // Store data at top of stack

POP Operation (memory stack):

DR ← M[SP] // Read data from top of stack SP ← SP + 1 // Increment SP
Note on growth direction: In many architectures, the memory stack grows DOWNWARD (from high addresses to low addresses). SP starts at a high address and decrements on push. This is the convention for x86, ARM, and the basic computer.

Uses of Stack

Storing subroutine return addresses, passing parameters to functions, saving register contents during interrupts, evaluating arithmetic expressions (postfix/reverse Polish notation).

Q2B Explain Subroutine / Define subroutine with example. [5 Marks]
3x Asked
▼
✅ Detailed Answer

A subroutine is a self-contained section of code that performs a specific, well-defined task. It can be called from multiple points in a program, avoiding code repetition.

Calling Convention in Basic Computer

The basic computer uses the BSA (Branch and Save Address) instruction to call subroutines:

ORG 200 LDA X BSA COMP / Call COMP subroutine — saves return addr at COMP, jumps to COMP+1 STA Y / Execution resumes HERE after return HLT X, DEC 25 Y, DEC 0 COMP, HEX 0 / This stores the return address (filled by BSA) CMA / Subroutine body: complement AC INC / Increment AC (result: 2's complement = negation) BUN COMP I / Indirect branch: PC ← M[COMP] = return address END

How BSA Works Step by Step

When BSA COMP executes: M[COMP] ← PC (saves the address of the NEXT instruction = 204 above), then PC ← COMP+1 (jumps into subroutine body). The subroutine runs. At BUN COMP I: it reads M[COMP]=204 and jumps back. Execution continues at STA Y.

Advantage of subroutines: Code reuse, modularity, smaller program size, easier debugging, allows separate development and testing of modules.

Q3B / Q4C Explain different Addressing Modes of instructions (any five). [5 Marks]
🔥 5x Asked
▼
✅ Detailed Answer

Addressing modes specify how the effective address (EA) of an operand is determined. Different modes offer trade-offs between code compactness, flexibility, and speed.

1. Implied (Inherent) Addressing

The operand is implied by the instruction itself — no explicit address field needed. The operand is always in a fixed location (like the AC register).

Example: CMA (Complement AC). The operand is always AC. No address needed.

2. Immediate Addressing

The operand itself is placed directly in the instruction (in the address field). No memory access needed for the operand. EA is not applicable — the value IS in the instruction.

Example: ADD #5 — adds 5 directly to AC. The number 5 is part of the instruction.

Advantage: Fast (no memory fetch for operand). Disadvantage: Limited range (only small constants).

3. Direct Addressing

The address field of the instruction contains the actual memory address of the operand. EA = address field.

Example: LDA 500 — EA=500, load M[500] into AC. One memory access needed.

4. Indirect Addressing

The address field contains the address of a memory location that holds the actual address (pointer). EA = M[address field]. Requires TWO memory accesses.

Example: LDA @500 — first fetch M[500]=700, then fetch M[700] = actual operand.

Advantage: Can address larger memory with a short address field. Useful for pointers.

5. Register Addressing

The operand is in one of the CPU's general-purpose registers. No memory access needed. Fast because registers are on-chip.

Example: ADD R2, R3 — EA = R3 (a register, not memory).

6. Indexed Addressing

EA = contents of Index Register (XR) + address field. Used for accessing array elements. By changing XR, you can traverse through an array systematically.

Example: LDA 500(XR) — if XR=10, EA=510. Change XR to 11 → EA=511.

7. Relative Addressing

EA = PC + address field (signed offset). Used in branch instructions. Makes code position-independent (relocatable). The offset specifies how far to jump from current PC.

Q3A / Q4A Draw and Explain Block Diagram of 64-word register stack with Full and Empty Flag. [5 Marks]
🔥 4x Asked
▼
✅ Detailed Answer

A Register Stack is a LIFO memory structure built entirely from registers inside the CPU. In a 64-word stack, there are 64 register locations, each 16 bits wide (for basic computer), numbered 0 to 63.

Components

Stack memory: 64 × 16-bit registers (words 0–63)
Stack Pointer (SP): 6-bit register, holds address of top of stack. Range: 000000 (0) to 111111 (63).
FULL flag: 1-bit flip-flop. Set to 1 when stack is full (cannot push anymore).
EMTY flag: 1-bit flip-flop. Set to 1 when stack is empty (cannot pop anymore).

PUSH Operation (Push DR onto stack)

SP ← SP + 1 // Increment stack pointer M[SP] ← DR // Store data register at top of stack If (SP = 63): FULL ← 1 // Stack is now full EMTY ← 0 // Stack is not empty

POP Operation (Pop stack into DR)

DR ← M[SP] // Load top of stack into data register SP ← SP - 1 // Decrement stack pointer If (SP = 0): EMTY ← 1 // Stack is now empty FULL ← 0 // Stack is not full

Error conditions: Pushing when FULL=1 is a stack overflow. Popping when EMTY=1 is a stack underflow. The hardware should check these before operations.

Q3B / Q4C Explain Three/Two/One/Zero address instructions with examples. [5 Marks]
🔥 5x Asked
▼
✅ Detailed Answer

Instructions can be classified by the number of explicit address fields they contain. This is the instruction format.

Three-Address Instructions

Format: OP D, S1, S2 — Destination + 2 sources. Most flexible.

Example: ADD R1, R2, R3 means R1 ← R2 + R3. The original values of R2 and R3 are preserved.

Advantage: Very expressive. Disadvantage: Long instructions (need bits for 3 addresses).

Evaluate X = (A+B)*(C+D):
ADD T1, A, B (T1←A+B)
ADD T2, C, D (T2←C+D)
MUL X, T1, T2 (X←T1*T2) — 3 instructions.

Two-Address Instructions

Format: OP D, S — Destination also serves as one source. Very common (x86, ARM).

Example: ADD R1, R2 means R1 ← R1 + R2. R1 is overwritten.

Evaluate X = (A+B)*(C+D):
MOV R1, A
ADD R1, B
MOV R2, C
ADD R2, D
MUL R1, R2
MOV X, R1 — 6 instructions.

One-Address Instructions

Format: OP A — Uses accumulator (AC) as implicit second operand/destination. This is the basic computer's architecture.

Example: ADD A means AC ← AC + M[A]. One memory operand, AC is implicit.

Evaluate X = (A+B)*(C+D):
LDA A, ADD B, STA T1, LDA C, ADD D, MUL T1, STA X — 7 instructions.

Zero-Address Instructions (Stack Machine)

Format: OP — No address field. All operands are on the stack. Push constants/variables onto stack; operations pop the top two values and push the result.

Evaluate X = (A+B)*(C+D):
PUSH A, PUSH B, ADD, PUSH C, PUSH D, ADD, MUL, POP X — 8 instructions.

Structure

The organization uses three internal buses: Bus A (source A), Bus B (source B), and Bus D (destination). The ALU receives inputs from Bus A and Bus B and outputs to Bus D.
